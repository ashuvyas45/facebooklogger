Add name to get the service name message

         

This shows the service name for all logging messages. This is useful for multiple loggers at different services.

Functions

  • let logs = new viablimp('TOKEN');
    • This returns a viablimp class object. The TOKEN should be obtained from facebook chatbot.
  • logs.message(loggingData)
    • This is a promise. When the promise is resolved, this returns the data as response status code. 200 for OK. 404 for not found. See response status codes for more info. This resolved promise data can be accessed by .then(data => {console.log(data)})
    If the token is incorrect or in case of no network, the promise gets rejected. That is it gets into .catch(error => {console.log(error)})
    Here loggingData can be anything to log. String, array, dictionary, etc.
  • logs.add_name('SERVICE_NAME')
    • Returns True. All log messages sent by calling from the viablimp object now will have the SERVICE_NAME as their heading as illustrated above.
